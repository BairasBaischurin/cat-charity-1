# QRKot — Приложение для Благотворительного фонда поддержки котиков

**QRKot** — это API-сервис для управления благотворительными проектами и сбора пожертвований. Сервис позволяет создавать целевые проекты по сбору средств (например, на медицинскую помощь или корм котикам) и принимать пожертвования от пользователей, которые автоматически распределяются по открытым проектам.

---

## Технологический стек

* **Python** 3.10+
* **FastAPI** — высокопроизводительный асинхронный веб-фреймворк
* **SQLAlchemy 2.0** — асинхронная ORM для работы с базой данных
* **Pydantic v2** — валидация данных и сериализация моделей
* **Alembic** — управление миграциями базы данных
* **SQLite** — используемая база данных (в режиме асинхронности через `aiosqlite`)
* **Pytest** — автоматическое тестирование бизнес-логики

## Как запустить проект локально

### 1. Клонирование репозитория и переход в папку
```bash
git clone git@github.com:BairasBaischurin/cat-charity-1.git
cd cat-charity-1
```

### 2. Настройка виртуального окружения
Создайте и активируйте виртуальное окружение, а затем обновите `pip`:
```bash
python -m venv venv
source venv/Scripts/activate  # Для Windows (Git Bash)
# venv\Scripts\activate      # Для Windows (PowerShell / CMD)
# source venv/bin/activate    # Для Linux / macOS
python -m pip install --upgrade pip
```

### 3. Установка зависимостей
Установите все необходимые пакеты для работы и тестирования проекта:
```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения
Создайте файл `.env` в корневой директории проекта и заполните его по шаблону:
```env
APP_TITLE=Благотворительный фонд поддержки котиков QRKot
APP_DESCRIPTION=Сервис для целевого сбора средств и распределения пожертвований.
DATABASE_URL=sqlite+aiosqlite:///./cat_charity.db
SECRET_KEY=SECRET_KEY_FOR_JWT_TOKENS_HERE
```

### 5. Применение миграций базы данных
Создайте структуру таблиц в базе данных с помощью Alembic:
```bash
alembic upgrade head
```

### 6. Запуск сервера разработки
Запустите приложение с помощью Uvicorn:
```bash
uvicorn app.main:app --reload
```
После запуска проект будет доступен по адресу: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## Документация API

После запуска сервера вам будут доступны интерактивные веб-интерфейсы для просмотра и тестирования эндпоинтов:
* **Swagger UI**: [http://127.0.0](http://127.0.0)
* **ReDoc**: [http://127.0.0](http://127.0.0)

### Основные эндпоинты:
* `POST /charity_project/` — Создание нового благотворительного проекта.
* `GET /charity_project/` — Получение списка всех проектов.
* `PATCH /charity_project/{project_id}` — Редактирование существующего проекта.
* `DELETE /charity_project/{project_id}` — Удаление проекта (если в него еще не инвестировали).
* `POST /donation/` — Сделать пожертвование (сумма автоматически инвестируется в старейший открытый проект).
* `GET /donation/` — Просмотр списка совершенных пожертвований.
